## Feature 1: Shields Ecosystem

### The Problem

Every user writes the same rules for Postgres, GitHub, and filesystem.
The rules for "safe Postgres usage" are identical across 1,000 teams.
We should ship those rules so users don't have to write them.

### What a Shield Is

A **Shield** is a named, versioned, pre-tested rule pack for one tool or service.

```typescript
interface Shield {
  name: string; // "postgres"
  version: string; // "1.2.0"
  description: string;
  author: string; // "node9" or "community"
  tools: string[]; // ["execute_query", "pg_query", "mcp__postgres__*"]
  rules: SmartRule[];
  ignoredTools: string[];
  testCases: ShieldTestCase[]; // self-tests: these inputs must be blocked/allowed
}
```

#### `postgres` Shield

```typescript
{
  name: 'postgres',
  rules: [
    { tool: '*', conditions: [{ field: 'sql', op: 'matches', value: /DROP\s+(TABLE|DATABASE|SCHEMA)/i }],  verdict: 'block',  reason: 'Destructive DDL' },
    { tool: '*', conditions: [{ field: 'sql', op: 'matches', value: /TRUNCATE\s+TABLE/i }],                verdict: 'block',  reason: 'Table truncation' },
    { tool: '*', conditions: [{ field: 'sql', op: 'matches', value: /ALTER\s+TABLE.*DROP\s+COLUMN/i }],    verdict: 'review', reason: 'Schema migration' },
    { tool: '*', conditions: [{ field: 'sql', op: 'matches', value: /DELETE\s+FROM\s+\w+(?!\s+WHERE)/i }], verdict: 'review', reason: 'DELETE without WHERE' },
    { tool: '*', conditions: [{ field: 'sql', op: 'matches', value: /UPDATE\s+\w+\s+SET(?!\s.*WHERE)/i }], verdict: 'review', reason: 'UPDATE without WHERE' },
    { tool: '*', conditions: [{ field: 'sql', op: 'matches', value: /^SELECT/i }],                        verdict: 'allow',  reason: '' },
    { tool: '*', conditions: [{ field: 'sql', op: 'matches', value: /^(EXPLAIN|SHOW|DESCRIBE)/i }],       verdict: 'allow',  reason: '' },
  ],
}
```

#### `github` Shield

```typescript
{
  name: 'github',
  rules: [
    { tool: 'bash', conditions: [{ field: 'command', op: 'matches', value: /git push.*--force.*(main|master|prod)/i }], verdict: 'block',  reason: 'Force push to protected branch' },
    { tool: 'bash', conditions: [{ field: 'command', op: 'matches', value: /git branch -[dD]\s+(main|master|prod)/i }], verdict: 'block',  reason: 'Deleting protected branch' },

    { tool: 'bash', conditions: [{ field: 'command', op: 'matches', value: /gh secret (set|delete)/i }],              verdict: 'review', reason: 'Modifying GitHub secrets' },

```

#### `bash-safe` Shield

```typescript
{
  name: 'bash-safe',
  rules: [
    { tool: 'bash', conditions: [{ field: 'command', op: 'matches', value: /(curl|wget).*\|\s*(bash|sh|zsh)/i }], verdict: 'block',  reason: 'Pipe to shell execution' },
    { tool: 'bash', conditions: [{ field: 'command', op: 'matches', value: /rm\s+-rf\s+[\/~]/i }],               verdict: 'block',  reason: 'Recursive delete from root/home' },
    { tool: 'bash', conditions: [{ field: 'command', op: 'matches', value: /mkfs\./i }],                         verdict: 'block',  reason: 'Filesystem format' },
    { tool: 'bash', conditions: [{ field: 'command', op: 'matches', value: /:()\s*\{/i }],                       verdict: 'block',  reason: 'Fork bomb pattern' },
    { tool: 'bash', conditions: [{ field: 'command', op: 'matches', value: /sudo\s+chmod\s+777\s+\//i }],        verdict: 'block',  reason: 'World-writable root' },
    { tool: 'bash', conditions: [{ field: 'command', op: 'matches', value: /python.*-c\s+['"]import/i }],        verdict: 'review', reason: 'Inline Python execution' },
  ],
}
```

### CLI Interface

```bash
node9 shield list                    # show available shields + status
node9 shield enable postgres         # activate (writes to config.json)
node9 shield disable postgres        # deactivate
node9 shield info postgres           # show rules in human-readable format
node9 shield test postgres           # run built-in test cases
node9 shield update                  # update all enabled shields to latest version
```

### Self-Tests

Every shield ships with test cases that verify correctness:

```typescript
// shields/postgres/tests.ts
export const testCases: ShieldTestCase[] = [
  { input: { tool: 'execute_query', args: { sql: 'DROP TABLE users' } }, expectedVerdict: 'block' },
  {
    input: { tool: 'execute_query', args: { sql: 'SELECT * FROM users' } },
    expectedVerdict: 'allow',
  },
  {
    input: { tool: 'execute_query', args: { sql: 'DELETE FROM users' } },
    expectedVerdict: 'review',
  },
  {
    input: { tool: 'execute_query', args: { sql: 'DELETE FROM users WHERE id = 1' } },
    expectedVerdict: 'allow',
  },
];
```

`node9 shield test postgres` runs all test cases and reports pass/fail.
CI runs `node9 shield test --all` to verify no shield is broken by policy engine changes.

## Feature 2: Org-Wide Dashboard (SaaS Mission Control)

### What It Is

The node9Firewall SaaS product gains team-level views for CISOs and security leads.

```
Team Dashboard
  ├── Live Feed: all AI tool calls across all developers in the org (right now)
  ├── Risk Summary: blocked/reviewed/allowed breakdown by developer, tool, project
  ├── Policy Studio v2: visual rule builder (no JSON)
  ├── Shield Manager: enable/disable shields for the whole org
  └── Audit Export: CSV/JSONL export for SIEM integration
```

### Team Risk Feed

```
DEVELOPER    TOOL           STATUS    RISK    TIME
alice        execute_query  🛑 block  HIGH    14:32:01  DROP TABLE sessions
bob          write_file     ✅ allow  LOW     14:32:05  /src/components/Button.tsx
carol        bash           🔶 review MED     14:32:09  curl api.internal.com/data
alice        write_file     🛑 block  CRITICAL 14:32:15 /.env (tainted path)
```

CISOs can filter by developer, tool, risk level, time range.
Click any row → full context: args, taint chain, which rule fired.
